проект в итоге будет представлять собой аналитическую платформу для сбора, обработки и визуализации пользовательских событий

# требования:

- в коде не должно быть комментариев и docsrtings

- должны быть тайпхинты

- по возможности не использовать глобальные переменные

- использовать DI, чтобы потом можно было легко покрывать код тестами

- архитектура с разделением на слои: сервисы, репозитории, модели БД, ДТО, эндпоинты и так далее

- используемые технологии: Python 3.13 (asyncio), FastAPI, Kafka, ClickHouse, Docker compose для разворачивания, в будущем Grafana для визуализации данных из кликхауса

в будущем получится такая архитектура: событие -> бекенд -> кафка -> кликхаус -> графана
